# Algorithm

Before creating a program, I first made a programming algorithm that I wrote on paper, yes on paper. and after the algorithm is created then I design Pseudocode

Algorithm writing must consist of the following 3 parts:

  • Algorithm title : a section consisting of the algorithm name and an explanation (specification) of the algorithm. The name should be short and describe what the algorithm does.
  • Declaration: The section to define all names used in the program. This name can be in the form of assignment names, variables, types, procedures and functions.
  • Description: This section contains a description of the steps to solve the problem written using the notation that will be explained later. Algorithm writing itself does not depend on the programming language specifications and the ability of the computer that executes it. Algorithmic notation is not programming language notation, but algorithms can be translated into various programming languages.

A programming algorithm is a specific sequence of logical steps to solve a problem. It emphasizes a logical sequence of steps, which means that the algorithm must follow a certain sequence, and the steps should not be skipped. Another definition of algorithm is a sequence of logical steps in solving a problem that is arranged systematically.

# Pseudocode

Programming is the process of "teaching" the computer how to do something. although it is known as pseudocode, pseudocode is actually not a programming language, psudocode is a basic algorithm that is often forgotten, in making fullstack programs I use pseudocode to simplify the logic in the language, I will provide basic pseudocode to consumers, and consumers will provide the input they want , so that the program will be made according to consumers wishes. if the pseudocode has been reached then I will continue creating the Database logic.
